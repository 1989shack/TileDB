NAME

tiledb_generate_data - generates synthetic data for a particular array schema in a CSV or binary form

 

SYNOPSIS

tiledb_generate_data -w directory -A array_name -f filename -t filetype -n cell_num [ -s seed ]

 

DESCRIPTION

tiledb_generate_data generates sythetic data for a particular array schema in a CSV or binary form. On error, it prints a message on stderr and returns an error code (see EXIT STATUS).

The dataset essentially consists of a collection of cells, whose format complies with the schema of an array given as argument to option -A. This means that the given array must already be defined. The number of cells to be generated, as well as the seed to the random generator, are given as arguments to the program as well.

Data distribution
Cooridinates: The coordinate values are drawn uniformly at random from their corresponding dimension domains defined in the array schema.

Attribute values: The attribute values depend on their type. The real data values (i.e., float and double) are drawn uniformly at random from [0.0,1.0]. The int and int64_t data values are drawn uniformly
 at random from [0,max()], where max() is the maximum value of the corresponding type supported by the system that calls the function. Finally, the char values are drawn uniformly at random from ASCII domain [45,126], i.e., all the characters between and including '-' and '~'.

CSV file format
This is described in detail in tiledb_load_csv(1). The difference is that the generated data do not contain null values and deletions.

Binary file format
This is described in detail in tiledb_load_bin(1). The difference is that the generated data do not contain null values and deletions.

 

OPTIONS

-w directory, --workspace=directory
The path to the workspace folder where the TileDB array is stored. The path must exist, and the caller must have read and write permissions on it.

-A array_name, --array-name=array_name
The name of the array for which the data are generated exported. The array must be already defined.

-f filename, --filename=filename
The name of the output file that will store the generated data.

-t filetype, --filetype=filetype
The type of the generated file. It can be either csv (CSV file) or bin (binary file).

-n cell_num, --cell-num=cell_num
The number of cells to be generated.

-s seed, --seed=seed
The seed that will be used internally for the random generator. This is optional. If it is omitted, the default is the current time of day.

 

EXAMPLES

tiledb_generate_data \

    -w my_workspace/ \
    -A A \
    -f A.csv \
    -t csv \
    -n 10000 \
    -s  0

Creates CSV data for array A. The data consist of 10000 cells. A fixed seed 0 was used, e.g., for repeatability purposes.

tiledb_generate_data \
    -w my_workspace/ \
    -A A \
    -f A.bin \
    -t bin \
    -n 10000

Creates binary data for array A. The data consist of 10000 cells. The default current time of day is used as a seed (which means that a different dataset will be generated next time the program is executed with the same options).

 

EXIT STATUS

0
Successful execution
-1
Parser error
-2
Undefined array
-3
File operation failed
-7
TileDB initialization failed
-8
TileDB finalization failed
-14
Failed to open array
-15
Failed to close array
