NAME

tiledb_export_csv - exports the (binary) data of an array into a CSV file

 

SYNOPSIS

tiledb_export_csv -w directory -A array_name -f filename [ -a attribute_name_1,... ] [ -d dim_name_1,... ] [ -m mode ]

 

DESCRIPTION

tiledb_export_csv exports the (binary) data of an array into a CSV file. The documentation on the exported CSV format can be found in program tiledb_load_csv. On error, it prints a message on stderr and returns an error code (see EXIT STATUS).

The following conventions apply to the SYNOPSIS section:

bold text type exactly as shown
italic text replace with appropriate argument
[-a arg] any or all options within [ ] are optional

The user may also specify a subset of attributes and dimensions from the array to export. Also, he/she may specify any order of attributes and dimensions, as well as multiplicities.
  In case the user wishes to completely hide the dimensions or the attributes, he/she should include __hide as the only name in the dimension or attribute name list.

Finally, the user may specify whether the array cells should be exported in the stored order or in reverse.
   

OPTIONS

-w directory, --workspace=directory
The path to the workspace folder where the TileDB are stored. The path must exist, and the caller must have read and write permissions on it.

-A array_name, --array-name=array_name
The name of the array whose data will be exported. The array must be already defined.

-f filename, --filename=filename
The name of the CSV file.

-a attribute_name_1,..., --attribute-names=attribute_name_1,...
The attributes to be exported. Any order and multiplicities are allowed. To export no attribute, include __hide instead of a name list. To export all attributes, omit the option.

-d dim_name_1,..., --dim-names=dim_name_1,...
The coordinates to be exported. Any order and multiplicities are allowed. To export no coordinates, include __hide instead of a name list. To export all coordinates, omit the option.

-m mode, --mode=mode
There are two modes: normal and reverse. In the former, the cells are exported in the same order they are stored (i.e., the native order specified in the array schema). In the latter, the cells are exported in the reverse order from the one they are stored. If the option is omitted, the default is normal.

 

EXAMPLES

Assuming that the array has dimensions dim1,dim2 and attributes attr1,attr2,attr3
tiledb_export_csv \

    -w my_workspace/ \
    -A A \
    -f A.csv

Exports into A.csv all the cells of array A in the order they are stored, including all the coordinates and attributes.

tiledb_export_csv \

    -w my_workspace/ \
    -A A \
    -f A.csv \
    -d dim1 \
    -a attr1,attr2

Exports int A.csv all the cells of A in the order they are stored, including only the coordinates on the dim1 dimension and the values on the attr1,attr2 attributes.

tiledb_export_csv \

    -w my_workspace/ \
    -A A \
    -f A.csv \
    -d dim1 \
    -a attr1,attr2 \
    -m reverse

Same as above, but the cells are exported in the reverse order.

tiledb_export_csv \

    -w my_workspace/ \
    -A A \
    -f A.csv \
    -d __hide \
    -a attr1,attr2 


 Same as the second example, but no coordinate is exported.

tiledb_export_csv \

    -w my_workspace/ \
    -A A \
    -f A.csv \
    -d dim1 \
    -a __hide

Same as the second example, but no attribute value is exported.

tiledb_export_csv \

    -w my_workspace/ \
    -A A \
    -f A.csv \
    -d dim1 \
    -a attr1,attr2,attr1

Same as the second example, but now the attr1 values are shown twice (once before those of attr2 and once after).

 

EXIT STATUS

0
Successful execution
-1
Parser error
-2
Undefined array
-3
File operation failed
-7
TileDB initialization failed
-8
TileDB finalization failed
-14
Failed to open array
-15
Failed to close array
