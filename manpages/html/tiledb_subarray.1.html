NAME

tiledb_subarray - creates a new array with the same schema as the input array (or including a subset of the attributes in a potentially different order), conataining only the cells that lie in the input range.

 

SYNOPSIS

tiledb_subarray -w directory -A array_name -R result_name -r range [ -a attribute_name_1,... ]

 

DESCRIPTION

tiledb_subarray creates a new array with the same schema as the input array (or including a subset of the attributes in a potentially different order), conataining only the cells that lie in the input range. The range must be a hyper-rectangle that is completely contained in the dimension space. It is also given as a sequence of [low,high] pairs across each dimension. On error, it prints a message on stderr and returns an error code (see EXIT STATUS).

The following conventions apply to the SYNOPSIS section:

bold text type exactly as shown
italic text replace with appropriate argument
[-a arg] any or all options within [ ] are optional

 

OPTIONS

-w directory, --workspace=directory
The path to the workspace folder where the TileDB are stored. The path must exist, and the caller must have read and write permissions on it.

-A array_name, --array-name=array_name
The name of the array the subarray will be applied on. The array must be already defined.

-R result_name, --result-name=result_name
The name of the output array.

-r range, --range=range
The range given as a sequence of [low,high] bounds across each dimension. Each range bound must be a real number.

-a attribute_name_1,..., --attribute-names=attribute_name_1,...
The attributes to be included in the array schema of the output array. Any order is allowed. To export all attributes, omit the option.

 

EXAMPLES

Assuming that the array has dimensions dim1,dim2 with domains [0,100] and [0,200], respectively, and attributes attr1,attr2,attr3
tiledb_subarray \

    -w my_workspace/ \
    -A A \
    -R A_sub \
    -r 10,20,100,150 

Creates a new array A_sub with exactly the same schema as A, but only containing the cells within rectangle [10,20] (dim1) , [100,150]R (dim2).

tiledb_subarray \

    -w my_workspace/ \
    -A A \
    -R A_sub \
    -r 10,20,100,150 \
    -a attr1,attr2 

Same as the first example, but now A_sub contains only two attributes (attr1,attr2).

tiledb_subarray \

    -w my_workspace/ \
    -A A \
    -R A_sub \
    -r 10,20,100,150 \
    -a attr2,attr1

Same as the second example, but attr1 and attr2 appear in a different order in A_sub.

 

EXIT STATUS

0
Successful execution
-1
Parser error
-2
Undefined array
-3
File operation failed
-7
TileDB initialization failed
-8
TileDB finalization failed
-14
Failed to open array
-15
Failed to close array
