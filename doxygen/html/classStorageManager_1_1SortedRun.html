Detailed Description

Fragment cells are sorted using a traditional external sortin algorithm. This algorithm produces 'sorted runs', i.e., sorted sequences of cells, during its 'sort' phase. Subsequently, during a 'merge' phase, multiple runs are merged into a single one (potentially recursively). A SortedRun object stores information about a sorted run.

Member Function Documentation

void StorageManager::SortedRun::advance_cell ( size_t  cell_size)

Advances the offset in the segment by cell_size to point to the next, logical cell. The input cell_size indicates the size of the lastly investigated cell, so that the object knows how many bytes to "jump" in the segment. Note that cell_size also encompasses the size of the potential id(s) the cell may carry.
